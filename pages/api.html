LLM API 层

services/api/ 负责与 Anthropic API 的所有通信，包括流式调用、重试、prompt cache 管理和 beta 功能协商。

调用流程

① 构建请求参数
  • normalizeMessagesForAPI() — 规范化消息格式，处理 thinking block 约束
  • buildSystemPromptBlocks() — 将 System Prompt 数组转为带 cache_control 的 block 列表
  • toolToAPISchema() — 将工具定义转为 API 所需的 JSON schema 格式
  • 协商 beta headers（thinking、prompt caching、fast mode、effort 等）
② 发起流式请求
  • 调用 anthropic.beta.messages.stream()
  • 携带 anthropic-beta header 启用实验性功能
  • 支持多个 API provider：Anthropic 直连、AWS Bedrock、GCP Vertex、自定义 base URL
③ 处理流式事件
  • content_block_start — 新 block 开始（text / tool_use / thinking）
  • content_block_delta — 增量内容，实时 yield 给 Agent 循环
  • message_delta — stop_reason、usage 统计
  • message_stop — 请求完成，记录 token 用量和费用
④ 错误处理与重试
  • 网络错误、超时 → withRetry() 指数退避重试，最多 10 次
  • 529 过载 → 仅前台查询重试（最多 3 次），后台任务直接失败
  • 401 OAuth → 自动刷新 token 后重试
  • 主模型失败 → 降级到 fallbackModel

Prompt Cache 机制

Claude Code 对 System Prompt 做了精细的缓存控制，以最大化 cache hit 率，降低 API 成本：

策略说明
scope: 'global'静态段落（BOUNDARY 之前）跨 org 共享缓存，所有用户共用同一份缓存
scope: 'session'动态段落（BOUNDARY 之后）仅在同一 session 内复用
1h cache支持 1 小时长效缓存（beta feature），适合长会话
cache editingcached microcompact 模式下可直接修改缓存内容，避免 cache bust

System Prompt 被拆分为多个 block，每个 block 独立设置 cache_control。静态 block 命中全局缓存时，只需支付极低的 cache read token 费用。

Beta Features 协商

Claude Code 通过 anthropic-beta header 启用多个实验性功能，在每次请求时动态决定是否携带：

Header功能
interleaved-thinking-2025-05-14Extended thinking 模式
prompt-caching-2024-07-31Prompt cache
fast-mode-2025-XX-XXFast mode（更快响应，有配额）
task-budgets-2026-03-13Token 预算控制
context-management-2025-XX-XXAPI 侧上下文管理
afk-modeAFK 自主模式

重试逻辑

services/api/withRetry.ts 实现了完整的重试策略：

const DEFAULT_MAX_RETRIES = 10
const BASE_DELAY_MS = 500      // 基础延迟
const MAX_529_RETRIES = 3      // 529 过载最多重试次数
const FLOOR_OUTPUT_TOKENS = 3000  // 最小输出 token 保底值

重试策略要点：

  • 指数退避：每次重试延迟翻倍，加随机 jitter 避免惊群
  • 529 过载只对前台查询（用户正在等待）重试，后台任务（compact、summary 等）直接失败，避免放大服务端压力
  • Fast mode 被拒绝时自动触发冷却，降级到普通模式重试
  • OAuth 401 自动刷新 token 后重试一次

多 Provider 支持

通过 getAPIProvider() 动态选择 API provider，支持：

  • Anthropic 直连：默认，使用 ANTHROPIC_API_KEY
  • AWS Bedrock：通过 AWS credentials，模型 ID 映射为 Bedrock inference profile
  • GCP Vertex AI：通过 GCP credentials
  • 自定义 base URL：ANTHROPIC_BASE_URL 环境变量，支持代理或兼容 API
