System Prompt 系统

System Prompt 是塑造 Claude 行为的核心机制。Claude Code 将其拆分为多个模块化段落，通过缓存策略大幅降低 API 成本。

整体结构

System Prompt 由 getSystemPrompt()（constants/prompts.ts）构建，返回一个字符串数组，每个元素对应一个段落。最终发送给 API 时合并为完整的 system 字段。

整体分为两大部分，以 SYSTEM_PROMPT_DYNAMIC_BOUNDARY 为分界线：

部分内容缓存策略
静态部分（前）身份介绍、行为规范、工具使用指南、语气风格等scope: 'global'，跨 org 共享缓存
动态部分（后）环境信息、语言偏好、MCP 指令、会话特定引导等按 session 缓存或每轮重算

为什么要分界？ 静态部分内容对所有用户相同，可以在 Anthropic 服务端跨 org 共享 prompt cache，节省大量 cache creation token。动态部分包含用户/会话特定信息，不能跨用户共享。

静态段落组成

按顺序拼接：

① Intro — 身份介绍
  • getSimpleIntroSection()
  • "You are an interactive agent that helps users with software engineering tasks..."
  • 包含安全警告：不生成/猜测 URL，注入防御说明
② System — 基础行为规范
  • getSimpleSystemSection()
  • 输出格式（Markdown、monospace）、工具权限模式、system-reminder 标签说明、自动压缩说明、Hooks 说明
③ Doing Tasks — 任务执行规范
  • getSimpleDoingTasksSection()
  • 不超出需求范围、先读后改、不猜时间、遇错先诊断、安全编码（OWASP）
  • 代码风格：不加不必要的注释、不做投机性抽象、三行重复好过一个过早的抽象
④ Actions — 谨慎操作规范
  • getActionsSection()
  • 破坏性操作前确认、不绕过 safety check、遇到未知文件先调查再删除
⑤ Using Tools — 工具使用指南
  • getUsingYourToolsSection()
  • 优先用专用工具而非 Bash、并行调用无依赖工具、用 TaskCreate 管理进度
⑥ Tone — 语气与风格
  • getSimpleToneAndStyleSection() + getOutputEfficiencySection()
  • 不用 emoji、简洁直接、引用代码带文件路径和行号
⑦ Dynamic Sections — 动态段落 边界以下
  • Session 特定引导、环境信息（CWD/OS/model）、语言偏好、MCP 指令、Memory、Scratchpad 等
  • 通过 systemPromptSection() / DANGEROUS_uncachedSystemPromptSection() 管理

段落缓存机制

动态段落通过 systemPromptSections.ts 中的两个函数管理：

// 普通缓存段落：计算一次，/clear 或 /compact 后失效
systemPromptSection('memory', () => loadMemoryPrompt())

// 危险的非缓存段落：每轮重算，会破坏 prompt cache
DANGEROUS_uncachedSystemPromptSection(
  'mcp_instructions',
  () => getMcpInstructionsSection(mcpClients),
  'MCP servers connect/disconnect between turns'  // 必须说明原因
)

为什么 uncached 是"危险的"？ 每次值变化都会导致该段落之后的所有内容 cache miss，产生额外的 cache creation token 费用。因此非缓存段落必须有充分理由，且函数名强制要求写明原因。

所有动态段落在 resolveSystemPromptSections() 中并行 resolve，结果存入 session 级 cache，下次直接读取：

// 命中缓存直接返回，否则计算并存入
if (!s.cacheBreak && cache.has(s.name)) {
  return cache.get(s.name) ?? null
}
const value = await s.compute()
setSystemPromptSectionCacheEntry(s.name, value)
return value

Proactive 模式的特殊 Prompt

当 feature flag PROACTIVE 或 KAIROS 启用且 proactive 模式激活时，整个 System Prompt 会被替换为一个极简版本：

return [
  `\nYou are an autonomous agent. Use the available tools to do useful work.\n\n${CYBER_RISK_INSTRUCTION}`,
  getSystemRemindersSection(),
  await loadMemoryPrompt(),
  envInfo,
  getLanguageSection(settings.language),
  getScratchpadInstructions(),
  getProactiveSection(),
].filter(s => s !== null)

这是 Claude Code 的"自主模式"——去掉了所有面向交互用户的行为约束，只保留安全底线和环境信息，让 Agent 自主决策。
