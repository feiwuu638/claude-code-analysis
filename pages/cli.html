CLI 交互层

CLI 层负责解析用户输入、处理斜杠命令、渲染终端 UI。它是用户与 Agent 循环之间的桥梁。

程序入口

main.tsx 是整个程序的入口，基于 Commander.js 解析 CLI 参数。启动时并行触发多个预加载任务以降低冷启动延迟：

// 在主模块 import 之前作为副作用触发
startMdmRawRead()        // MDM 企业配置预读
startKeychainPrefetch()  // macOS Keychain 凭证预读
// + API 预连接、GrowthBook feature flag 初始化

参数解析完成后，调用 renderAndRun() 启动 React/Ink 渲染循环，进入交互模式。

斜杠命令系统

用户以 / 开头输入的命令由 commands.ts 统一注册和分发。源码中共有约 50 个斜杠命令：

Skill 命令： 用户自定义的 Skill 也以 /skill-name 形式调用，由 SkillTool 执行。Skill 本质上是一段 Markdown 格式的 prompt 模板，被展开后注入到对话中。

终端 UI 框架（Ink）

Claude Code 使用 React + Ink 构建终端 UI，src/ink/ 是对 Ink 的自定义封装，src/components/ 包含约 140 个 React 组件。

Ink 的核心思路是：用 React 的声明式模型描述终端界面，通过 Yoga 布局引擎计算 flexbox 布局，最终输出 ANSI 转义码渲染到终端。

子目录职责
ink/renderer.ts核心渲染器，将 React 树转为 ANSI 输出
ink/layout/Yoga flexbox 布局引擎集成
ink/events/键盘、鼠标事件处理
ink/termio/终端 I/O，ANSI 转义码处理
ink/hooks/终端相关 React hooks（useInput、useStdout 等）
components/对话气泡、工具调用展示、权限弹窗、进度条等 UI 组件

权限模式与用户交互

在默认模式下，当 Claude 尝试调用危险工具（如写文件、执行命令）时，终端会弹出权限确认框，用户可以选择：

  • 允许一次 — 仅本次允许
  • 始终允许 — 将该工具加入白名单
  • 拒绝 — 拒绝并告知 Claude

用户也可以通过 --dangerously-skip-permissions 启动参数跳过所有权限检查，进入完全自主模式。

Hooks 系统

用户可以在 settings.json 中配置 hooks，在特定事件发生时执行 shell 命令：

// 示例：每次工具调用前执行
{
  "hooks": {
    "PreToolUse": [{ "command": "echo 'tool called'" }],
    "PostToolUse": [{ "command": "notify-send 'done'" }],
    "SessionStart": [{ "command": "./bootstrap.sh" }]
  }
}

Hook 的输出会作为 <user-prompt-submit-hook> 标签注入到对话中，Claude 会将其视为来自用户的反馈。
