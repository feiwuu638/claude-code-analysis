工具系统（Tools）

Claude Code 的能力边界由工具决定。每个工具是一个自包含模块，定义了输入 schema、权限模型、执行逻辑和给 Claude 的使用说明（prompt）。

工具模块结构

每个工具位于 src/tools/{ToolName}/，通常包含：

文件职责
prompt.ts工具描述 prompt，告诉 Claude 这个工具是什么、什么时候用、参数怎么填
{ToolName}.ts工具实现：输入 schema（Zod）、执行逻辑、权限检查
constants.ts工具名称常量及共享配置
UI.tsx权限确认对话框的 React 组件（部分工具有）

工具基类型定义在 src/Tool.ts，所有工具实现该接口，包括：输入 schema、call() 执行方法、renderToolUseMessage() UI 渲染、权限模式配置等。

工具分类

文件操作类

工具说明
Read读取文件，支持图片、PDF、Jupyter notebook，最多 2000 行，可指定 offset/limit
Write创建或完整覆写文件
Edit精确字符串替换，必须先 Read 才能 Edit，old_string 必须唯一
Glob按 glob 模式匹配文件路径，结果按修改时间排序
Grep基于 ripgrep 的内容搜索，支持正则、文件类型过滤、多种输出模式
NotebookEdit编辑 Jupyter notebook 的指定 cell

执行类

工具说明
Bash在用户 shell 环境执行命令，支持超时设置和后台执行（run_in_background）

Web 类

工具说明
WebFetch抓取 URL 内容，HTML 转 Markdown，用小模型按 prompt 提取信息，带 15 分钟缓存
WebSearch网页搜索，返回结构化搜索结果

Agent 类

工具说明
Agent启动子 Agent 处理复杂多步任务，支持指定 subagent_type、后台运行、worktree 隔离
SendMessage向已有 Agent 发送消息，Agent 保持完整上下文继续执行

任务管理类

工具说明
TaskCreate创建任务，用于追踪多步工作进度
TaskUpdate更新任务状态（pending/in_progress/completed/deleted）
TaskGet获取任务详情
TaskList列出所有任务

工作流类

工具说明
EnterPlanMode进入 Plan 模式，Claude 只能规划不能执行
ExitPlanMode退出 Plan 模式，等待用户审批
EnterWorktree创建 git worktree 隔离工作环境
ExitWorktree退出 worktree，选择保留或删除
AskUserQuestion向用户提问，提供选项列表
Skill执行用户定义的 Skill（可复用工作流）

MCP / 扩展类

工具说明
MCPTool调用外部 MCP server 提供的工具
ListMcpResources列出 MCP server 的资源列表
ReadMcpResource读取 MCP server 的指定资源
RemoteTrigger调用远程触发器 API
ScheduleCron创建定时触发任务

权限系统

每次工具调用前，都会经过 hooks/toolPermission/ 的多层权限检查。权限模式由 AppState.toolPermissionContext.mode 控制：

模式行为
default危险操作弹窗询问用户，安全操作自动允许
plan只允许读操作，写/执行操作全部拒绝
bypassPermissions跳过所有权限检查，自动允许（需用户显式开启）
auto非交互模式，无法弹窗，根据规则自动决策

权限检查通过三种 handler 实现，对应不同运行场景：

  • interactiveHandler — 普通交互会话，可弹窗询问用户
  • swarmWorkerHandler — 子 Agent，继承父 Agent 的权限配置
  • coordinatorHandler — 多 Agent 协调器，统一管理权限

关键工具 Prompt

每个工具的 prompt.ts 是告诉 Claude 如何使用该工具的说明书，直接影响 Claude 的工具调用行为。
