整体架构

Claude Code 是一个分层的 Agentic 系统。本章从顶层视角梳理各模块的职责边界与相互关系。

目录结构

源码全部位于 src/ 下，按功能划分为以下主要目录：

src/
├── main.tsx              # 程序入口，CLI 解析 + React/Ink 渲染初始化
├── query.ts              # Agent 主循环（核心）
├── QueryEngine.ts        # LLM 查询引擎，流式响应处理
├── Tool.ts               # 工具基类型定义
├── tools.ts              # 工具注册表
├── commands.ts           # 斜杠命令注册
│
├── tools/                # 40+ 个工具实现（每个工具一个目录）
├── commands/             # 斜杠命令实现（~50 个）
├── components/           # Ink UI 组件（~140 个）
├── hooks/                # React hooks（含权限系统）
├── services/             # 外部服务集成
│   ├── api/              # Anthropic API 客户端
│   ├── mcp/              # MCP 协议
│   ├── compact/          # 上下文压缩
│   ├── analytics/        # GrowthBook feature flags + 遥测
│   └── ...
├── utils/                # 工具函数（150+ 子目录）
├── constants/            # 常量，含 System Prompt 构建逻辑
├── types/                # TypeScript 类型定义
├── state/                # 全局状态管理
├── bridge/               # IDE 扩展双向通信（VS Code / JetBrains）
├── coordinator/          # 多 Agent 协调器
├── ink/                  # 自定义终端 UI 框架
├── memdir/               # 持久化 Memory 目录
├── skills/               # Skill 系统
└── query/                # 查询管道子模块

关键设计模式

并行预加载（Startup 优化）

启动时在主模块加载前并行触发多个预加载任务，显著降低冷启动延迟：

// main.tsx — 作为副作用在 import 之前触发
startMdmRawRead()       // MDM 配置预读
startKeychainPrefetch() // Keychain 凭证预读
// + API 预连接、GrowthBook 初始化

Feature Flag 死代码消除

通过 Bun 的 bun:bundle 在构建时静态消除未启用的功能代码，外部用户构建中这些代码完全不存在：

import { feature } from 'bun:bundle'

const voiceCommand = feature('VOICE_MODE')
  ? require('./commands/voice/index.js').default
  : null  // 外部构建中此分支被完全删除

已知的 Feature Flag 包括：PROACTIVE、KAIROS、BRIDGE_MODE、DAEMON、VOICE_MODE、AGENT_TRIGGERS、MONITOR_TOOL、TOKEN_BUDGET、HISTORY_SNIP、CONTEXT_COLLAPSE 等。

懒加载重模块

OpenTelemetry、gRPC、分析模块等重量级依赖通过动态 import() 延迟加载，只在真正需要时才初始化。
