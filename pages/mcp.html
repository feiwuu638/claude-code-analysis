MCP 集成

Model Context Protocol（MCP）是 Anthropic 推出的开放标准，让 Claude Code 能够连接外部服务、数据库、API，动态扩展工具能力。

什么是 MCP

MCP 定义了一套标准协议，让 AI 模型可以通过统一接口调用外部工具（Tools）、读取外部资源（Resources）、使用预设提示词（Prompts）。Claude Code 作为 MCP 客户端，连接用户配置的 MCP server，将其工具动态注入到 Agent 的工具集中。

传输协议

MCP server 支持多种传输方式：

传输类型说明适用场景
stdio通过标准输入输出与子进程通信本地命令行工具（最常用）
sseServer-Sent Events，HTTP 长连接远程 HTTP server
httpStreamable HTTP（新标准）现代远程 server
wsWebSocket需要双向实时通信
sdk直接 SDK 集成内置 MCP server

配置方式

MCP server 在 settings.json 中配置，支持多个作用域：

// ~/.claude/settings.json 或项目 .claude/settings.json
{
  "mcpServers": {
    "my-server": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@my-org/my-mcp-server"],
      "env": { "API_KEY": "..." }
    },
    "remote-server": {
      "type": "sse",
      "url": "https://my-server.example.com/sse"
    }
  }
}

配置作用域优先级（高到低）：enterprise → managed → project → user → local

连接生命周期

① 启动连接
  • connectToServer() 根据传输类型创建对应 transport
  • stdio server：fork 子进程，通过 stdin/stdout 通信
  • HTTP/SSE server：建立 HTTP 连接，处理认证（OAuth / API Key）
② 能力协商
  • fetchToolsForClient() 获取 server 提供的工具列表
  • 获取 server 的 instructions（注入 System Prompt 的 MCP 指令段落）
  • 获取可用资源列表（Resources）
③ 工具注入
  • MCP 工具被包装为 MCPTool 实例，加入 Agent 的工具集
  • 工具名称格式：mcp__serverName__toolName
  • Server instructions 注入 System Prompt 的 mcp_instructions 段落
④ 工具调用
  • Claude 调用 MCP 工具时，MCPTool.call() 通过 MCP 协议转发给 server
  • server 执行并返回结果（文本、图片、资源引用等）
  • 支持流式进度更新（MCPProgress）

认证机制

MCP server 支持多种认证方式：

  • OAuth 2.0：支持标准 OAuth 流程，token 存储在 macOS Keychain，自动刷新
  • Cross-App Access（XAA）：企业级跨应用授权，通过 IdP 统一管理
  • API Key：通过 env 配置传入环境变量
  • 无认证：本地 stdio server 通常不需要认证

Resources 与 Prompts

除工具外，MCP server 还可以提供：

  • Resources：可读取的数据源（文件、数据库记录、API 响应等），通过 ListMcpResources 和 ReadMcpResource 工具访问
  • Prompts：预设提示词模板，可通过 /mcp 命令查看和使用

MCP 管理命令

用户可通过 /mcp 斜杠命令管理 MCP server：

  • 查看已连接的 server 列表和状态
  • 查看每个 server 提供的工具
  • 重新连接断开的 server
  • 查看 server 提供的 Prompts

MCP 的架构意义： MCP 将 Claude Code 从一个固定能力的工具变成了一个可无限扩展的平台。任何人都可以编写 MCP server，为 Claude 提供访问私有数据库、内部 API、专有工具的能力，而无需修改 Claude Code 本身。
